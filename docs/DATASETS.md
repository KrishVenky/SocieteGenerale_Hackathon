# Datasets

## Primary: PS4 Official Sample Data

**Location**: `data/Problem_04_Data_Access/sample_data/`
**Source**: Provided by Societe Generale hackathon organisers for PS4

### data_access_logs.csv

1,201 events across Apr 2025 – Apr 2026 (full calendar year). 100 users.

**Actual column schema**:

```
timestamp            datetime    When the access happened
user_id              string      USR00000–USR00099
username             string      firstname.lastname
action               category    export_data | sql_query | api_call |
                                 admin_operation | login | file_access
resource             category    Customer_Vault | GL_System | HRIS | PROD_DB |
                                 Admin_Console | SIEM | Data_Lake |
                                 BI_Tool | File_Share | Email_Archive
resource_sensitivity category    low | medium | high
status               category    success | failure
source_ip            string      192.168.x.x (internal range)
time_classification  category    business_hours | night | unusual_hours | weekend
```

**No `rowcount`, `destination`, or `anomaly_marker` in actual data** , README described an idealized schema that differs from the real CSV.

### user_profiles.csv

100 users. Actual column schema:

```
user_id          string      USR00000–USR00099
username         string      firstname.lastname
email            string      @company.com
department       category    Finance | Security | IT | Marketing | Engineering |
                             Compliance | Legal | Sales | HR | Operations |
                             Support | Executive
job_title        string      Developer | Director | Architect | Engineer | etc.
privilege_level  category    user | power-user | admin | service-account
systems_access   string      Pipe-separated approved systems:
                             AD | Azure_AD | SIEM | EMAIL | PROD_DB |
                             GCP | Okta | Salesforce | ServiceNow |
                             AWS_IAM | VPN | ADMIN_SYS
last_login       date
days_inactive    integer     Days since last login (high risk if > 30)
is_active        bool
hire_date        date
```

**No pre-computed `typical_access_hours`, `avg_queries_per_day`, or `avg_rowcount_per_query`.**
Behavioral baselines are derived from log history, not from this profile file.

### Ground Truth (Labels)

`data_access_labels.csv` and `user_profile_labels.csv` referenced in the README were **not included** in the sample data ZIP. They may be:
- Released separately at evaluation time
- Expected to be generated by our system

**Our label strategy**: inject 4 labeled anomaly events (the demo personas), evaluate against those + any naturally detectable anomalies. If official labels are released, plug them in immediately.
